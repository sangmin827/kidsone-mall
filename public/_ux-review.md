## 다음 세션에서 이어갈 항목 (심각도 순)

### 상 (Blocking UX)

1. ⏳ **결제 플로우 — 비회원 분기 UX**
   결제 완료 후 `/guest-order/result` 가 searchParams(orderNumber, phone) 의존 → 새로고침 시 정보 손실. 주문완료 직후에는 세션스토리지 or 별도 `/guest-order/complete?token=xxx` 로 토큰 발급하는 구조로 개선 권장.

2. ⏳ **카카오 Postcode 스크립트 로딩 실패 폴백**
   현재는 토스트 에러만 띄움. 수동 입력 가능한 text input 필드로 폴백 제공 필요.

3. ⏳ **로그인/회원가입 흐름**
   소셜 로그인만 제공 → 이메일/비번 옵션 부재. 최소한 비회원도 `/guest-order` 로 우회 가능한 안내 배너가 필요.

### 중 (경험 악화)

4. ⏳ **로딩 상태 (Suspense / 스켈레톤) 전면 누락**
   `/products`, `/mypage/orders`, `/mypage/cart`, `/checkout` 모두 서버 렌더라서 느린 네트워크에서 blank 화면. 최소한 `loading.tsx` 파일을 각 라우트에 추가.

5. ⏳ **모바일 터치 타겟**
   장바구니 수량 -/+ 버튼이 9×9 (h-9 w-9) → iOS HIG 권장 44×44 미만. `h-10 w-10` + touch-target padding 권장.

6. ⏳ **품절 상태 UI**
   현재 버튼 disabled 처리만. `품절` 뱃지(빨간 태그)를 상품 이미지 위에 덮어주고 회색조 필터 적용 권장.

7. ⏳ **주문 조회/마이페이지 빈 상태 디자인**
   텍스트 한 줄만 표시. 아이콘 + CTA 버튼(예: "상품 보러가기") 있으면 전환율↑.

### 하 (개선사항)

8. ⏳ **홈 랜딩 콘텐츠 부족**
   히어로 + 주요 상품 섹션 + Top 10 프리뷰 + 공지사항 배너 등 섹션 구조 필요.

9. ⏳ **접근성**
   - 모달/드롭다운에 focus trap 없음
   - 이미지 `alt` 는 상품명으로 잘 들어가 있으나, 배경 이미지는 `aria-hidden` 필요
   - 색 대비: 플레이스홀더 `text-gray-400` → WCAG AA 기준 미달 가능성

10. ⏳ **/categories/[slug] 페이지에도 ProductsToolbar 재사용**
    정렬/검색 기능이 카테고리 페이지에는 빠져 있음. 재사용 가능하게 리팩토링.

## 권장 작업 순서

1. 결제 완료 토큰 기반으로 전환 (상 #1) — 데이터 손실 방지 먼저
2. 각 주요 라우트에 `loading.tsx` 추가 (중 #4) — 체감 속도 크게 개선됨
3. 품절 뱃지 + 모바일 터치 타겟 정리 (중 #5, #6) — 하루 작업
4. 카카오 Postcode 폴백 + 로그인 분기 안내 (상 #2, #3)
5. 홈 랜딩 재설계 (하 #8)
6. 접근성 패스 (하 #9)
